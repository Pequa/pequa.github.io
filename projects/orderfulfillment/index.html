ORDER FULFILLMENT

BEFORE: ALL OVER THE PLACE

Our existing Control Panel, specifically the Recent Orders screen shown above, is where dealers see all of the incoming orders that need to be fulfilled and shipped. This screen dates back to the early days of the company and has been in desperate need of a re-design. However, this project was two-fold. The other functionality we wanted to add was PDC warehouse fulfillment, where dealers could submit items they didn’t have on-hand from a customer’s order to one of the manufacturer’s warehouses, which would ship out the items to the dealer in ready-to-ship packaging to send to the customer.

AFTER: ALL-IN-ONE

The old screen allowed the user to do two things: receive/ship the order, or go to the Order Details screen for more information and to complete the rest of the steps needed to process an order. The Order Details screen is just as dated, with 10 tabs that contain an overload of information laid out in an antiquated way. This new screen was designed to show all of the information a dealer would need about an order, and all of the functionality to fulfill a typical order, or submit the order for PDC fulfillment. Access to the order details screen remains for edge-case orders, but the new screen presents much more information, laid out for a much better, more robust user experience.

NEW DESIGN LANGUAGE

These mockups were the first screen I re-designed in the Control Panel since we began switching to a new design language. So as part of this project, I laid out a framework for styling similar pages, so that as we move forward with the rest of the Control Panel, design language can maintain consistency.
